# 10. 服务管理器

## 10.1 简介

服务管理器（Service Manager）是一个基于类型索引和命名索引的服务定位器（Service Locator）单例类，采用线程安全的读写锁，支持按接口类型注册/获取服务，或按命名注册/获取服务。服务实例以 std::shared_ptr 形式管理，自动处理生命周期，并可选择性地支持 ServiceBase 接口的优雅关闭。

## 10.2 特性

* **类型安全**：通过模板接口按类型注册和获取服务，无需手动转换；
* **线程安全**：使用 std::shared_mutex 实现读多写少场景下的高效并发；
* **支持命名服务**：同一接口类型可注册多个不同名称的实例（如“数据库A”、“数据库B”）；
* **自动清理**：服务继承自 ServiceBase 时，服务管理器析构时自动调用所有服务的关闭函数（若服务继承自 ServiceBase）；
* **轻量无侵入**：服务无需继承任何基类即可注册（但若需要自动关闭，建议继承 ServiceBase）；
* **编译期检查**：对继承 ServiceBase 的服务进行虚析构函数静态断言。

## 10.3 使用方法

### 10.3.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Service Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	ServiceManager::get().init();
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取单例
core::ServiceManager& sm = core::ServiceManager::get();
// 初始化
bool isSucceeded = sm.init();
```

这将清空所有已注册服务，返回 true，通常在程序启动时调用。

### 10.3.2 注册服务

使用 registerService() 为接口类型注册一个普通服务（单例）。

```cpp
class ILogger
{
public:
    virtual void log(const std::string& msg) = 0;
    virtual ~ILogger() = default;
};

class ConsoleLogger : public ILogger
{
public:
    void log(const std::string& msg) override
    {
        std::cout << msg << std::endl;
    }
};

auto logger = std::make_shared<ConsoleLogger>();
sm.registerService<ILogger>(logger);
```

注意：每个接口能且只能注册一个服务，如果同一接口类型重复注册，会记录警告并忽略后续注册。

使用 registerNamedService() 同一接口类型可注册多个不同名称的实例，此时允许同一接口注册多个服务。

```cpp
class IDatabase
{
public:
    virtual void connect() = 0;
    virtual ~IDatabase() = default;
};

class MySQLDatabase : public IDatabase
{
public:
    void connect() override
    {
        std::cout << "Connecting to MySQL\n";
    }
};

class PostgreSQLDatabase : public IDatabase
{
public:
    void connect() override
    {
        std::cout << "Connecting to PostgreSQL\n";
    }
};

sm.registerNamedService<IDatabase>("mysql", std::make_shared<MySQLDatabase>());
sm.registerNamedService<IDatabase>("postgres", std::make_shared<PostgreSQLDatabase>());
```

如果服务类继承自 core::ServiceBase（定义在 virServiceBase.hpp），Service Manager 在注册时会自动存储一个关闭函数，在 shutdown() 时调用该服务的 shutdown() 方法。

ServiceBase 定义为：

```cpp
#pragma once
#include "defCoreApi.hpp"

#include <string>

namespace core
{
    class ServiceBase
    {
    public:
        virtual ~ServiceBase() = default;

        virtual bool init() { return true; }
        virtual void shutdown() {}
        virtual const std::string& getName() const = 0;
    };
}
```

对于非 ServiceBase 派生的服务，shutdownFunc 为空，不会自动调用任何清理操作。

### 10.3.3 获取服务

使用 getService() 获取普通服务，这将返回指定接口的服务实例，若未注册返回 nullptr。

```cpp
auto logger = sm.getService<ILogger>();
if (logger)
{
    logger->log("Hello, ServiceManager!");
}
```

使用 getNamedService() 获取命名服务，这将返回指定接口和名称的服务实例，若不存在返回 nullptr。

```cpp
auto mysql = sm.getNamedService<IDatabase>("mysql");
if (mysql) {
    mysql->connect();
}
```

### 10.3.4 注销服务

使用 unregisterService() 注销普通服务，这将移除指定接口类型的服务。

```cpp
template <typename Interface>
void unregisterService();
```

使用 unregisterNamedService() 注销命名服务，这将移除指定接口类型和名称的服务。

```cpp
template <typename Interface>
void unregisterNamedService(const std::string& name);
```

注意：考虑到注销服务时可能会有其他的对象调用此服务，为了避免未定义的行为出现，注销服务不会调用其 shutdownFunc()，只是从管理器中移除引用。实际对象的销毁将由 shared_ptr 的引用计数决定。

### 10.3.5 清理与关闭

使用 shutdown() 清理申请的资源，即遍历所有命名服务和普通服务，如果服务存在 shutdownFunc() ，则此函数将被调用以清空内部容器。

```cpp
sm.shutdown();
```

析构函数自动调用 shutdown()，确保资源释放。

## 10.4 完整示例

服务定义

```cpp
// example.hpp
// 可选基类（若需自动关闭）
#include "virServiceBase.hpp"

// 假设 ICalculator 继承自 ServiceBase 基类
class ICalculator : public core::ServiceBase
{
public:
    virtual int add(int a, int b) = 0;
    virtual ~ICalculator() = default;

};

class SimpleCalculator : public ICalculator
{
public:
    int add(int a, int b) override
    {
        return a + b;
    }

    void shutdown() override
    {
        std::cout << "Calculator shutting down..." << std::endl;
    }

};

class IFormatter
{
public:
    virtual std::string format(const std::string& input) = 0;
    virtual ~IFormatter() = default;

};

class UpperFormatter : public IFormatter
{
public:
    std::string format(const std::string& input) override
    {
        std::string result = input;
        for (char& c : result) c = toupper(c);
        return result;
    }

};

class LowerFormatter : public IFormatter
{
public:
    std::string format(const std::string& input) override
    {
        std::string result = input;
        for (char& c : result) c = tolower(c);
        return result;
    }

};
```

```cpp
// 使用 ServiceManager
#include "Service/lgcServiceManager.hpp"
#include "example.hpp"
#include <iostream>

int main()
{
    auto& sm = core::ServiceManager::get();
    sm.init();

    // 注册普通服务
    sm.registerService<ICalculator>(std::make_shared<SimpleCalculator>());

    // 注册命名服务（同一接口多实例）
    sm.registerNamedService<IFormatter>("upper", std::make_shared<UpperFormatter>());
    sm.registerNamedService<IFormatter>("lower", std::make_shared<LowerFormatter>());

    // 获取并使用服务
    auto calc = sm.getService<ICalculator>();
    if (calc)
    {
        std::cout << "3 + 5 = " << calc->add(3, 5) << std::endl;
    }

    auto upper = sm.getNamedService<IFormatter>("upper");
    if (upper)
    {
        std::cout << upper->format("Hello World") << std::endl; // 输出: HELLO WORLD
    }

    auto lower = sm.getNamedService<IFormatter>("lower");
    if (lower)
    {
        std::cout << lower->format("Hello World") << std::endl; // 输出: hello world
    }

    // 注销示例
    sm.unregisterNamedService<IFormatter>("lower");
    // lower 对象此时若无其他引用则被销毁

    // 程序退出时自动调用 shutdown，触发 SimpleCalculator::shutdown()
    return 0;
}
```

## 10.5 线程安全

所有 registerXxx() 、 getXxx() 、 unregisterXxx() 操作均使用读写锁保护。

读操作（getService() 、getNamedService()）可并发执行。

写操作（registerXxx() 、 unregisterXxx() 、 shutdown()）独占锁，保证数据一致性。

## 10.6 注意事项
* 接口要求：注册的服务实例类型必须可转换为 std::shared_ptr<Interface>。若使用命名服务，同一接口类型的名称必须唯一（内部键为 typeid(Interface).name() + "::" + name）。建议接口类（如 ILogger）具有虚析构函数，确保通过基类指针删除派生对象时的正确性；
* 关于 requireService：代码中注释掉了一个 requireService 模板方法，若需要获取必需服务（不存在时抛出异常），可自行取消注释或添加类似功能；
* 服务生命周期：服务实例以 shared_ptr 形式存储，外部也可持有 shared_ptr，注意避免循环引用。调用 shutdown() 时，shutdownFunc 执行完毕后，管理器才释放对服务的引用（通过 clear()）。因此 shutdown() 函数内部不应再调用 getService 获取自身或其他可能已被清理的服务，否则可能导致未定义行为；
* 虚析构函数检查：对于继承 ServiceBase 的服务，代码中有静态断言确保通过基类指针删除时正确调用派生类析构函数：

```cpp
static_assert(std::has_virtual_destructor<Interface>::value,
    "ServiceBase derived classes must have a virtual destructor!");
```

* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 线程安全：虽然 Service Manager 本身线程安全，但服务实例内部可能非线程安全，使用时需注意同步；
* 性能：typeid(Interface).name() 返回值是编译器相关的，但作为键足够稳定。读操作（获取服务）使用共享锁，适合高频获取、低频注册的场景。
