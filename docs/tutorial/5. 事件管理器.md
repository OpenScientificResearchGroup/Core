# 5. 事件管理器

## 5.1 简介

事件管理器（Event Manager）是一个基于主题（Topic）和事件类型的事件分发单例类，支持按主题订阅/发布事件，使用 std::any 存储异构事件处理器，通过读写锁保证线程安全。

## 5.2 特性

* **主题驱动**：支持按主题分类事件（如 "user_login"、"file_saved"），空字符串 "" 表示全局主题；
* **类型安全**：事件类型通过模板参数指定，订阅者只接收匹配类型的事件；
* **多订阅者**：同一主题同一事件类型可注册多个回调，每个回调分配唯一 ID；
* **线程安全**：使用 std::shared_mutex 实现读多写少场景的高效并发；
* **灵活订阅**：支持带主题和不带主题的订阅/发布重载；
* **排除发送者**：发布时可指定 excludedId，避免回调自己（防止循环）；
* **自动 ID 管理**：订阅返回唯一 ID，可用于取消订阅。

## 5.3 使用方法

### 5.3.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Event Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	EventManager::get().init();
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取单例
core::EventManager& evtMgr = core::EventManager::get();
// 初始化
bool isSucceeded = evtMgr.init();
```

这将清空所有订阅者和 ID 映射，重置 ID 计数器。通常在程序启动时调用。

### 5.3.2 订阅事件

使用 subscribe() 订阅一个事件，这将返回一个唯一 ID，可用于后续取消订阅。通过第一个参数是直接传入回调函数还是传入主题名区分订阅的是特定主题事件还是全局主题事件。

```cpp
struct FileSavedEvent {
    std::string filePath;
    bool success;
};

struct UserLoginEvent {
    std::string userName;
    int userId;
};

// 订阅特定主题
size_t id1 = evtMgr.subscribe<FileSavedEvent>("editor", [](const FileSavedEvent& e) {
    std::cout << "Editor: File " << e.filePath 
              << (e.success ? " saved" : " save failed") << std::endl;
});

// 订阅全局主题
size_t id2 = evtMgr.subscribe<UserLoginEvent>([](const UserLoginEvent& e) {
    std::cout << "User logged in: " << e.userName << " (ID: " << e.userId << ")" << std::endl;
});
```

### 5.3.3 发布事件

发布事件到指定主题或全局主题。可指定 excludedId 排除某个订阅者（通常是发送者自身防止回环）。同样通过第一个参数是直接传入回调函数还是传入主题名区分发布的是特定主题事件还是全局主题事件。

```cpp
// 发布到 "editor" 主题
FileSavedEvent saveEvent{"document.txt", true};
evtMgr.publish("editor", saveEvent);

// 发布到全局主题
UserLoginEvent loginEvent{"alice", 1001};
evtMgr.publish(loginEvent);

// 发布时排除自己（假设当前组件 ID 为 id1）
evtMgr.publish("editor", saveEvent, id1);
```

### 5.3.4 取消订阅

使用 unsubscribe() 取消订阅，取消订阅时需要传入订阅时返回的 ID 。需要指定事件类型以正确查找处理器。

```cpp
evtMgr.unsubscribe<FileSavedEvent>(id1);
evtMgr.unsubscribe<UserLoginEvent>(id2);
```

### 5.3.5 清理与关闭

使用 shutdown() 清空所有数据，其效果同 init()。析构函数将会自动调用 shutdown() 。

```cpp
evtMgr.shutdown();
```

## 5.4 完整示例

定义事件结构

```cpp
// EventType.hpp
// 事件可以是任意可复制类型（建议简单结构）
struct TextChangedEvent {
    std::string newText;
    int cursorPos;
};

struct WindowResizedEvent {
    int width;
    int height;
};

struct ButtonClickedEvent {
    int buttonId;
    std::string buttonLabel;
};
```

使用 EventManager

```cpp
#include "Event/lgcEventManager.hpp"
#include "EventType.hpp"
#include <iostream>
#include <thread>

int main()
{
    auto& evtMgr = core::EventManager::get();
    evtMgr.init();

    // 订阅者1：监听全局文本变化
    size_t textSubId = evtMgr.subscribe<TextChangedEvent>([](const TextChangedEvent& e) {
        std::cout << "[Global] Text changed to: \"" << e.newText 
                  << "\" at cursor " << e.cursorPos << std::endl;
    });

    // 订阅者2：只监听 "editor" 主题的文本变化
    size_t editorTextSubId = evtMgr.subscribe<TextChangedEvent>("editor", [](const TextChangedEvent& e) {
        std::cout << "[Editor] Text updated: \"" << e.newText << "\"" << std::endl;
    });

    // 订阅者3：监听窗口大小变化
    size_t windowSubId = evtMgr.subscribe<WindowResizedEvent>([](const WindowResizedEvent& e) {
        std::cout << "Window resized to " << e.width << "x" << e.height << std::endl;
    });

    // 发布事件
    TextChangedEvent textEvt{"Hello World", 5};
    evtMgr.publish(textEvt);                 // 全局主题，两个订阅者都会收到
    evtMgr.publish("editor", textEvt);       // 仅 "editor" 主题订阅者收到

    WindowResizedEvent winEvt{1024, 768};
    evtMgr.publish(winEvt);                   // 全局主题

    // 取消订阅
    evtMgr.unsubscribe<TextChangedEvent>(textSubId);

    // 再次发布，已取消的订阅者不再接收
    evtMgr.publish(textEvt);                   // 仅 editor 订阅者收到

    // 发布时排除某个 ID（例如发送者自己）
    evtMgr.publish(winEvt, windowSubId);       // 排除 windowSubId，即 window 订阅者不会收到

    evtMgr.shutdown();
    return 0;
}
```

多线程示例

```cpp
// multithread.cpp
#include "Event/lgcEventManager.hpp"
#include "EventType.hpp"

void publisherThread()
{
    auto& evtMgr = core::EventManager::get();
    for (int i = 0; i < 10; ++i)
    {
        TextChangedEvent evt{"Thread msg " + std::to_string(i), i};
        evtMgr.publish(evt);
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
}

void subscriberThread()
{
    auto& evtMgr = core::EventManager::get();
    size_t id = evtMgr.subscribe<TextChangedEvent>([](const TextChangedEvent& e) {
        std::cout << "Received: " << e.newText << std::endl;
    });
    std::this_thread::sleep_for(std::chrono::seconds(2));
    evtMgr.unsubscribe<TextChangedEvent>(id);
}
```

## 5.5 线程安全
读操作（publish()）使用共享锁，允许多个线程同时发布事件。

写操作（subscribe() 、 unsubscribe() 、 init() 、 shutdown()）使用独占锁。

回调函数的执行在锁内进行（publish 遍历时调用用户回调），因此回调应避免长时间阻塞，且不应调用会修改订阅关系的函数（可能导致死锁或迭代器失效）。

## 5.6 注意事项
* 事件类型要求：事件类型可以是任何可复制的类型，但是事件对象需在发布期间保持有效。若事件需跨线程传递，建议使用值语义的小型结构，或使用 shared_ptr 包装；
* 回调执行位置：回调在 publish 调用线程中同步执行。若回调耗时较长，会阻塞发布者。如需异步处理，应在回调内部将事件转发到工作队列；
* 死锁风险：回调函数中不要调用 subscribe() / unsubscribe() / publish() 等可能加锁的操作，否则可能导致死锁（因为 publish 已持有共享锁，再次加写锁会阻塞）。若确需在回调中修改订阅关系，可延迟处理（如通过队列）；
* 类型安全：订阅和取消订阅必须使用相同的事件类型，否则取消会由于查找不到对应处理器而失败。unsubscribe 不会报错，仅静默失败；
* 排除 ID：excludedId 用于防止发送者收到自己的事件（常见于 UI 控件避免循环）。排除比较在遍历时进行，性能损耗极小；
* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 性能考虑：事件分发是 O(主题数 + 处理器数)，适合中等规模的订阅者。若需极高吞吐量（如每秒百万级事件），可考虑改用无锁队列或通道（Channel）模式。
