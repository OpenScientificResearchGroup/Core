# 12. 界面管理器

## 12.1 简介

界面管理器（Ui Manager）是一个基于 wxWidgets 的 UI 管理单例类，负责主窗口生命周期、AUI 布局管理、控件锚点注册与查找、控件 ID 的字符串映射以及布局的持久化（JSON）。它通过读写锁保证读多写少场景下的线程安全。

## 12.2 特性
* **主窗口管理**：创建并持有 wxFrame，提供显示/关闭接口；
* **AUI 集成**：管理 wxAuiManager，支持布局的保存与加载；
* **锚点系统**：将任意 wxObject*（窗口、菜单、工具栏等）注册到字符串键，支持自动清理；
* **ID 映射**：通过字符串名称获取整数 ID（自动生成），支持匿名 ID 分配；
* **布局持久化**：将窗口大小、最大化状态、AUI perspective 保存/加载为 JSON 文件；
* **线程安全**：锚点和 ID 映射使用 std::shared_mutex，读操作并发，写操作独占。

## 12.3 使用方法

### 12.3.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Ui Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	UiManager::get().init(appName);
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取 Ui 管理器
UiManager& ui = UiManager::get();
// 主窗口名称
std::string title = "Test";
// 主窗口宽度
size_t width = 800;
// 主窗口高度
size_t height = 600;
bool isSucceeded = ui.init(title, width, height);
```

在初始化时，会自动注册主窗口（wxFrame*）和 AUI 管理器（wxAuiManager*）的锚点，分别为“/Core/main_frame”和“/Core/aui_manager”。

init() 返回 true 表示成功。

### 12.3.2 显示与关闭窗口

```cpp
ui.show();   // 显示主窗口
ui.close();  // 关闭主窗口
```

### 12.3.3 获取核心对象

通过调用 getRootWindow() 和 getAuiManager() 直接获取。

```cpp
wxFrame* frame = ui.getRootWindow();            // 获取主窗口指针
wxAuiManager* auiMgr = ui.getAuiManager();      // 获取 AUI 管理器指针
```

### 12.3.4 锚点管理

锚点机制允许将任意 wxObject*（窗口、菜单、工具栏、图像列表等）与一个字符串键关联，便于在程序任何地方快速查找。

使用 registerAnchor() 注册锚点并将 obj 关联到 anchorId：
```cpp
std::string anchorId = "/main/panel";
wxPanel* panel = new wxPanel(ui.getRootWindow(), wxID_ANY);
ui.registerAnchor(anchorId, panel);
```

如果 obj 是 wxWindow 派生类，自动绑定 wxEVT_DESTROY 事件，窗口销毁时自动移除锚点（避免野指针）。

使用 unregisterAnchor() 注销锚点，这样可以手动移除锚点，但是通常不需要，因为自动清理已处理：

```cpp
// 假设 panel 已经不需要了
std::string anchorId = "/main/panel";
ui.unregisterAnchor(anchorId);   // 上一步注册的 panel 将会被移除
```

使用 getAnchor() 获取锚点，这将返回关联的 wxObject*，若不存在返回 nullptr：

```cpp
// 现在要获取刚刚的 panel
std::string anchorId = "/main/panel";
wxPanel* panel = dynamic_cast<wxPanel*>(getAnchor(anchorId)); // 使用 dynamic_cast 确保类型是我们需要的
```

如果不想显式使用 dynamic_cast 确保类型安全，这里也有类型安全的获取，这将自动转型为指定类型，失败返回 nullptr：

```cpp
// 现在要获取刚刚的 panel
std::string anchorId = "/main/panel";
wxPanel* panel = ui.getAnchorAs<wxPanel>(anchorId);
```

使用 refreshAnchorLayout() 强制更新界面布局：
```cpp
// 现在要强制刷新刚刚的 panel
std::string anchorId = "/main/panel";
ui.refreshAnchorLayout(anchorId);
```

如果锚点是 wxWindow，将调用其 Layout()、Refresh()、Update()。

如果锚点是 wxSizer，将调用其 Layout()。

### 12.3.5 ID 管理

界面管理器中内置 ID 管理器，将字符串名称与整数 ID 相互映射，避免在代码中硬编码 ID 值。

使用 getId() 获取 ID（自动分配），如果名称已存在，返回已有 ID；否则分配新 ID 并存储映射:

```cpp
// 现在想要创建一个按键，并且添加到刚刚的 panel 上
size_t btnId = ui.getId("ClickButton");
wxButton* btn = new wxButton(panel, btnId, "Click Me");
```

使用 removeId() 移除 ID ，这将删除名称与 ID 的映射，并返回是否成功。

```cpp
// 移除刚刚的按键 ID
std::string name = "ClickButton";
bool isSucceeded = ui.removeId(name);
```

创建匿名 ID
cpp
size_t createAnonymousId();
分配一个未与任何名称关联的 ID（用于临时控件）。

根据 ID 获取名称
cpp
std::string getName(size_t id) const;
返回 ID 对应的名称，若不存在返回 "Anonymous_or_Unknown"。

典型用法：

cpp
size_t idSave = ui.getId("FileSave");
wxMenuItem* item = new wxMenuItem(menu, idSave, "&Save\tCtrl+S");
// 事件表中使用 idSave

### 12.3.6 布局持久化

使用 saveLayoutToJson() 保存当前的界面布局到 JSON ：

```cpp
// 我想关闭界面了，但是又想重现这个布局
std::string path = "layout.json";
ui.saveLayoutToJson(path);
```

保存的 JSON 格式将会长这个样子：

```json
{
    "app_name": "App",
    "version": "1.0",
    "window": {
        "width": 1024,
        "height": 768,
        "maximized": false
    },
    "layout_perspective": "..."
}
```

使用 loadLayoutFromJson() 从 JSON 加载布局，这将读取 JSON 文件，恢复窗口状态和 AUI 布局，若文件不存在或解析失败，仅记录错误，不改变当前布局（不出意外应该是回到默认布局）：

```cpp
// 重现刚刚保存的布局
std::string path = "layout.json";
ui.loadLayoutFromJson(path);
```

一般来说，我们希望能在程序启动时能重现上一次的布局，并且在程序退出时保存布局，于是就可以这样调用：

```cpp
// 程序启动时
ui.loadLayoutFromJson("layout.json");
// 程序退出时
ui.saveLayoutToJson("layout.json");
```

### 12.3.7 清理与关闭

清理界面管理器时，请调用 shutdown() 。

```cpp
ui.shutdown();
```

这将销毁 AUI 管理器，清空所有锚点和 ID 映射。

## 12.4 完整示例
```cpp
#include "Ui/lgcUiManager.hpp"
#include <wx/wx.h>

class MyApp : public wxApp
{
public:
    virtual bool OnInit() override
    {
        // 初始化 UiManager
        auto& ui = core::UiManager::get();
        if (!ui.init("Demo App", 800, 600))
            return false;

        // 创建自定义面板并注册为锚点
        wxPanel* panel = new wxPanel(ui.getRootWindow(), wxID_ANY);
        ui.registerAnchor("/main/panel", panel);

        // 创建按钮，使用 ID 管理
        size_t btnId = ui.getId("ClickButton");
        wxButton* btn = new wxButton(panel, btnId, "Click Me");

        // 绑定事件
        Bind(btnId, &MyApp::OnClick, this);

        // 加载上次布局
        ui.loadLayoutFromJson("layout.json");

        // 显示主窗口
        ui.show();
        return true;
    }

    virtual int OnExit() override
    {
        // 保存布局
        core::UiManager::get().saveLayoutToJson("layout.json");
        return wxApp::OnExit();
    }

    void OnClick(wxCommandEvent&)
    {
        wxMessageBox("Hello from UiManager!");
    }
};

wxIMPLEMENT_APP(MyApp);
```

## 12.5 线程安全说明

锚点映射和 ID 映射均使用 std::shared_mutex 保护。

读操作（如 getAnchor、getId 的首次读）可并发执行。

写操作（如 registerAnchor、removeId）独占锁，保证数据一致性。

注意：wxWidgets 本身要求 UI 操作在主线程执行，因此锚点相关的 UI 刷新应确保在主线程调用。

## 12.6 注意事项

* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 锚点自动清理：仅对 wxWindow 派生类有效，其他类型（如 wxMenu、wxToolBar）需手动注销或在适当时候置空；
* ID 线程安全：getId 是线程安全的，但 wxWidgets 的事件处理应在主线程进行；
* 布局文件版本：代码中固定了 app_name 和 version，可根据实际需求调整。
