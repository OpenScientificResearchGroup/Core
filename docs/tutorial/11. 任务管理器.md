# 11. 任务管理器

## 11.1 简介

任务管理器（Task Manager）是一个基于线程池的任务调度管理器，支持任务优先级、取消机制（通过 CancellationToken）以及 std::future 返回结果。它采用单例模式，内部使用优先队列管理任务，多个工作线程并发执行。

## 11.2 特性

* **线程池**：指定数量的工作线程（默认使用硬件并发数）。
* **任务优先级**：支持 LOW、NORMAL、HIGH、EMERGENCY 四级优先级。
* **可取消任务**：通过 CancellationToken 允许在任务执行前或执行中取消。
* **返回结果**：提交任务返回 std::future 获取返回值或异常。
* **自动停止**：析构或调用 shutdown 时会等待所有任务完成。

## 11.3 必须的类

CancellationToken 类用于请求和检查任务取消的令牌，基于 std::shared_ptr<std::atomic\<bool\>>，支持多线程安全复制，其定义如下：

```cpp
#pragma once
#include "defCoreApi.hpp"

#include <memory>
#include <atomic>

namespace core
{
	class CORE_API CancellationToken
	{
	public:
		CancellationToken();

		// 请求取消任务
		void cancel() const;

		// 检查任务是否已被请求取消
		[[nodiscard]] bool isCancellationRequested() const;

		// 重载 bool 操作符，方便在 if 语句中使用
		explicit operator bool() const;

	private:
		/// <summary>
		/// 当您在 submit 函数中创建 token 时，会创建一个新的 shared_ptr，指向一个新的 atomic<bool>
		/// 当这个 token 对象（及其底层的 shared_ptr）被复制到其他地方（例如，作为参数传递给任务执行函数），shared_ptr 的引用计数会增加
		/// 这意味着： 只要还有一个 CancellationToken 对象的副本存在，并且它指向的原子变量的引用计数大于零，取消状态就会保持有效
		/// 只要没有所有的令牌副本都被销毁，状态就不会丢失
		/// 这使得令牌可以安全地在多个线程间传递和共享
		/// </summary>
		std::shared_ptr<std::atomic<bool>> mIsCancelled;
	};
}
```

## 11.4 使用方法

### 11.4.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Task Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	TaskManager::get().init(ConfigManager::get().getValue<size_t>("/Core/task/num_threads"));
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取 Task 管理器
auto& tm = core::TaskManager::get();
// 使用 4 个工作线程
size_t numThreads = 4;
bool isSucceeded = tm.init(numThreads);
```

在程序启动时调用 init，指定线程数量。若未指定或传入 0，则将线程数设定为当前系统能够承载的最大线程数，如果仍然小于等于0，将会被设置为1。

### 11.4.2 提交任务

如果任务函数接受 CancellationToken 作为第一个参数，则视为提交了可取消任务。提交后返回 std::pair<std::future\<ReturnType\>, CancellationToken>，其中 future 用于获取结果，token 可用于外部请求取消。

```cpp
auto [fut, token] = tm.add(
    core::Priority::NORMAL,
    [](core::CancellationToken token, int a, int b) -> int
    {
        for (int i = 0; i < 10; ++i)
        {
            if (token)
            {  // 检查是否被取消
                std::cout << "任务被取消\n";
                throw std::runtime_error("Task cancelled at " + std::to_string(i));
            }
            std::this_thread::sleep_for(std::chrono::milliseconds(100));
        }
        return a + b;
    },
    3, 4 // 参数 a, b
    );

// 可以在其他线程中取消
std::thread([&]{
    std::this_thread::sleep_for(std::chrono::milliseconds(200));
    token.cancel();
    }).detach();

// 获取结果
try
{
    int result = fut.get(); // 若被取消，此处抛出异常
    std::cout << "Result: " << result << std::endl;
}
catch (const std::exception& e)
{
    std::cerr << "Task failed: " << e.what() << std::endl;
}
```

如果函数签名不接受 CancellationToken，则视为提交了不可取消任务。提交后返回 std::future<ReturnType>。

```cpp
auto fut = tm.add(
    core::Priority::HIGH,
    [](int a, int b) { return a * b; },
    5, 6
    );
int result = fut.get();  // 30
```

任务优先级定义在 core::Priority 枚举中，从低到高为：LOW、NORMAL、HIGH、EMERGENCY。

优先级高的任务会被优先执行（最大堆）。注意，如果提交速度持续超过处理速度，低优先级任务可能延迟较久，造成饥饿现象。

### 11.4.3 取消任务

取消操作通过 CancellationToken 的 cancel() 方法触发。取消是一个“请求”而非强制终止，任务代码需要定期检查 token.isCancellationRequested() 或使用 token.operator bool() 来响应取消。

如果任务尚未开始执行（仍在队列中），执行时会在进入任务函数前检查 token，若已取消则抛出异常。

在任务执行过程中，应定期检查 token，以便及时响应取消请求。

```cpp
auto task = [](core::CancellationToken token) {
    while (!token.isCancellationRequested()) {
        // 执行一部分工作
        // 如果 token 变为 true，退出循环
    }
    // 清理或返回默认值
};
```

### 11.4.4 清理与关闭

程序退出前调用 shutdown() ，会通知所有工作线程停止，并等待已提交任务完成。析构函数会自动调用 shutdown() 。

```cpp
tm.shutdown();
```

## 11.5 完整示例

```cpp
#include "Task/lgcTaskManager.hpp"
#include <iostream>
#include <thread>

int main()
{
    auto& tm = core::TaskManager::get();
    tm.init();  // 使用默认线程数

    // 提交一个可取消的任务
    auto [fut1, token1] = tm.add(core::Priority::HIGH,
        [](core::CancellationToken token, int start) -> std::string
        {
            for (int i = 0; i < 10; ++i)
            {
                if (token)
                    throw std::runtime_error("Task cancelled at " + std::to_string(i));
                std::this_thread::sleep_for(std::chrono::milliseconds(50));
            }
            return "Completed";
        },
        0);

    // 提交一个不可取消的任务
    auto fut2 = tm.add(core::Priority::NORMAL, [](int x) { return x * x; }, 7);

    // 1 秒后取消第一个任务
    std::this_thread::sleep_for(std::chrono::seconds(1));
    token1.cancel();

    // 获取结果
    try
    {
        std::string result = fut1.get(); // 若被取消，此处抛出异常
        std::cout << "Result: " << result << std::endl;
    }
    catch (const std::exception& e)
    {
        std::cerr << "Task failed: " << e.what() << std::endl;
    }

    // 获取第二个任务的结果（不会抛出异常）
    std::cout << "Task2 result: " << fut2.get() << std::endl;

    tm.shutdown();
    return 0;
}
```

## 11.6 线程安全

TaskManager 是单例，add 方法内部对任务队列加锁，可多线程安全调用。

CancellationToken 的 cancel 和 isCancellationRequested 是线程安全的（原子操作）。

工作线程在任务执行期间不会持有队列锁，因此任务内部可安全地再次提交新任务。

## 11.7 注意事项

* 可取消任务的返回值类型：当前代码在取消时直接当检测到取消时，将会抛出异常。
* 任务队列大小限制：TaskManager 未限制任务队列长度，如果提交速度持续高于处理速度，可能导致内存无限增长，可根据需要添加最大队列容量。
* 饥饿：任务管理器采用了优先队列，默认优先级越高的任务越先执行，如果持续提交高优先级任务，低优先级任务可能延迟较久，造成饥饿现象。
* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 异常安全：若任务函数抛出异常，异常会通过 std::future 传播给 get() 调用者，任务执行期间的异常不会影响线程池稳定性；
* 线程局部存储：如果任务需要使用线程局部存储，请注意线程池中的线程会被复用，因此 TLS 变量会保持之前任务的值，需手动重置。
