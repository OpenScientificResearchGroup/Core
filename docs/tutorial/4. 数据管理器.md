# 4. 数据管理器

## 4.1 简介

数据管理器（Data Manager）是一个基于文档模型的数据管理单例类，负责管理多个文档的生命周期、激活状态和全局访问。每个文档必须继承自 ContainerBase，提供 UUID、名称、时间戳等基础功能。

## 4.2 特性

* **文档管理**：创建、关闭、获取所有文档；
* **激活状态**：跟踪当前活动文档，支持切换；
* **自动命名**：新文档自动生成名称（如 Untitled-1）；
* **唯一标识**：每个文档自动分配 UUID；
* **线程安全**：使用 std::shared_mutex 实现读多写少场景的高效并发。

## 4.3 必须的基类

数据管理器需要 ContainerBase 类，任何想要加载在数据管理器中的资源必须继承这个类并在其基础上拓展业务功能，例如：

```cpp
#pragma once
#include <Data/virContainerBase.hpp>
#include <Command/lgcCommandManager.hpp>

class Document : public core::ContainerBase
{
public:
    Document();
    virtual ~Document() = default;

    bool load(const std::string& path);		
    bool save(const std::string& path = "");

    core::CommandManager& getCommandManager();
private:
    core::CommandManager mCommandManager; // 命令管理器，用于处理撤销/重做等操作
};
```

## 4.4 使用方法

### 4.4.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Data Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
    DataManager::get().init();
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取单例
core::DataManager& dataMgr = core::DataManager::get();
// 初始化
bool isSucceeded = dataMgr.init();
```

这将清空所有在内存中的文档和清空活动文档在内存中的 UUID，并将未命名计数器设为 1。

### 4.4.2 创建文档

使用 createDocument() 创建一个文档，这将返回新创建的文档智能指针，并自动设置为活动文档。

```cpp
// 假设有 Document 类继承自 ContainerBase
class MyDocument : public core::ContainerBase
{
    // 实现接口...
};

auto doc1 = dataMgr.createDocument<MyDocument>(); // doc1 的名称为 "Untitled-1"
```

### 4.4.3 关闭文档

使用 closeDocument() 关闭一个文档，这将关闭指定 UUID 的文档。

```cpp
// 假设我们知道待关闭文档的 UUID
std::string uuid;
bool isSucceeded = dataMgr.closeDocument(uuid);
```

注意：如果关闭的是活动文档，会自动将另一个文档设为活动（如有）。

使用 closeAllDocuments() 关闭所有文档，这将清空所有在内存中的文档和清空活动文档在内存中的 UUID。

```cpp
dataMgr.closeAllDocuments();
```

### 4.4.4 文档查询

使用 getActiveDocument() 获取活动文档，这将返回当前活动文档的基类指针，若无活动文档返回 nullptr 。

```cpp
std::shared_ptr<MyDocument> docPtr = dataMgr.getActiveDocument();
```

### 4.4.5 获取指定文档

使用 getDocument() 根据 UUID 获取文档，不存在返回 nullptr。

```cpp
// 假设我们知道待获取文档的 UUID
std::string uuid;
std::shared_ptr<MyDocument> docPtr = dataMgr.getDocument(uuid);
```

### 4.4.6 获取所有文档

使用 getAllDocuments() 获取所有的文档，这将返回所有文档的列表（顺序不定）。

```cpp
std::vector<std::shared_ptr<MyDocument>> docs = dataMgr.getAllDocuments();
```

### 4.4.7 设置活动文档

使用 setActiveDocument() 将指定 UUID 的文档设为活动文档。若 UUID 为空或不存在，则不改变当前活动文档；若传入的 UUID 与当前相同，则不做任何操作。

```cpp
// 假设我们还知道待激活文档的 UUID
std::string uuid;
dataMgr.setActiveDocument(uuid);
```

### 4.4.8 清理与关闭

使用 shutdown() 清理与关闭 Data Manager，这将调用 closeAllDocuments() 关闭所有文档，并且析构函数中会自动调用 shutdown() 。

```cpp
dataMgr.shutdown();
```

## 4.5 完整示例

```cpp
// Document.hpp
#pragma once
#include <Data/virContainerBase.hpp>
#include <Command/lgcCommandManager.hpp>

class Document : public core::ContainerBase
{
public:
    Document();
    virtual ~Document() = default;

    bool load(const std::string& path);		
    bool save(const std::string& path = "");

    core::CommandManager& getCommandManager();
private:
    core::CommandManager mCommandManager; // 命令管理器，用于处理撤销/重做等操作
};
```

```cpp
#include "Data/lgcDataManager.hpp"
#include "Document.hpp"
#include <iostream>

int main()
{
    auto& dm = core::DataManager::get();
    dm.init();

    // 创建几个文档
    auto doc1 = dm.createDocument<Document>("Document");
    doc1->setContent("This is document 1");

    auto doc2 = dm.createDocument<Document>("Document");
    doc2->setContent("This is document 2");

    // 获取所有文档
    auto allDocs = dm.getAllDocuments();
    std::cout << "Total documents: " << allDocs.size() << std::endl;

    // 获取活动文档（应该是 doc2，因为最后创建）
    auto active = dm.getActiveDocument();
    if (active)
    {
        std::cout << "Active document: " << active->getName() << std::endl;
    }

    // 切换到 doc1
    dm.setActiveDocument(doc1->getUuid());

    // 关闭 doc2
    dm.closeDocument(doc2->getUuid());

    // 再次获取活动文档（应该是 doc1）
    active = dm.getActiveDocument();
    if (active)
    {
        std::cout << "Active document after close: " << active->getName() << std::endl;
    }

    // 关闭所有文档
    dm.closeAllDocuments();

    return 0;
}
```

## 4.6 线程安全

读操作（getActiveDocument() 、 getDocument() 、 getAllDocuments()）使用共享锁，可并发执行。

写操作（createDocument() 、 closeDocument() 、 closeAllDocuments() 、 setActiveDocument()）使用独占锁。

注意：文档对象内部的操作（如 setName()）不在 Data Manager 的锁保护范围内，需文档自身保证线程安全。

## 4.7 注意事项
* 文档类型要求：文档必须继承自 ContainerBase，否则编译期静态断言会报错。
* UUID 生成：在 Utility/lgcUuid.hpp 中有 UUID 生成函数。
* 文档名称计数器：mUntitledCount 是原子类型，但仅在 createDocument 中递增。若多个线程同时创建文档，名称后缀可能不连续但唯一。
* 文档生命周期：DataManager 持有文档的 shared_ptr，外部也可持有。文档在最后一个 shared_ptr 销毁时自动释放。关闭文档仅从管理器中移除引用，若外部仍有 shared_ptr，文档对象仍存活。
