# 9. 资源管理器

## 9.1 简介

资源管理器（Resource Manager）是一个基于虚拟文件系统（VFS）的资源管理单例类，提供资源的挂载、加载器注册、缓存、重新加载和自动卸载功能。它采用读写锁保证线程安全，支持通过逻辑路径（如 assets://textures/hero.png）访问物理文件。

## 9.2 特性

* **虚拟文件系统**：通过别名（Alias）将逻辑路径映射到物理路径，如 assets:// → /data/assets/。
* **扩展加载器**：按文件扩展名注册自定义加载函数（如 .png、.json、.obj）。
* **自动缓存**：已加载资源缓存，避免重复磁盘 I/O。
* **线程安全**：使用 std::shared_mutex 实现读多写少场景的高效并发。
* **资源重载**：支持重新加载单个资源（热更新）。
* **垃圾回收**：unloadUnused() 释放无外部引用的缓存资源。
* **类型安全**：模板接口 getRes<T> 返回指定类型的资源指针。

## 9.3 必须的基类

资源管理器需要 ResourceBase 类，任何想要加载在资源管理器中的资源必须继承这个类并实现相关的函数，定义如下：

```cpp
#pragma once
#include "defCoreApi.hpp"

#include <string>

namespace core
{
	class CORE_API ResourceBase
	{
	public:
		virtual ~ResourceBase() = default;

		// 获取资源名称/路径
		const std::string& getName() const { return mName; }
		// 获取资源占用的内存大小（用于统计）
		virtual size_t getSizeInBytes() const = 0;

	protected:
		friend class ResourceManager;
		void setName(const std::string& name) { mName = name; }
		std::string mName;

	};
}

```

## 9.4 使用方法

### 9.4.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Resource Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	ResourceManager::get().init();
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取单例
core::ResourceManager& resMgr = core::ResourceManager::get();
// 初始化
bool isSucceeded = resMgr.init();
```

这将清空所有挂载点、加载器和缓存。通常在程序启动时调用。

### 9.4.2 挂载虚拟路径

使用 mount() 将逻辑别名映射到物理目录：

```cpp
// 挂载 assets 别名到程序目录下的 assets 文件夹
std::string alias = "assets";
std::filesystem::path path = "./assets"
resMgr.mount(alias, path);
// 之后可通过 "assets://textures/hero.png" 访问
```

注意：别名会被规范化，例如 "assets://" → "assets" ， "assets:" → "assets" ， "assets" → "assets" ，此时物理路径会转换为绝对路径存储。

### 9.4.3 注册资源加载器

每种文件扩展名需要注册对应的加载函数，使用 registerLoader() 注册。

```cpp
// 纹理加载器
resMgr.registerLoader(".png", [](const std::filesystem::path& path) -> std::shared_ptr<ResourceBase> {
    auto texture = std::make_shared<TextureResource>();
    if (texture->loadFromFile(path))
        return texture;
    return nullptr;
});

// JSON 加载器
resMgr.registerLoader(".json", [](const std::filesystem::path& path) -> std::shared_ptr<ResourceBase> {
    auto jsonRes = std::make_shared<JsonResource>();
    if (jsonRes->parse(path))
        return jsonRes;
    return nullptr;
});
```

### 9.4.4 获取资源

使用 getRes() 通过逻辑路径获取资源。若资源已缓存则直接返回，否则调用对应加载器加载并缓存。如果路径包含 ://，提取别名（如 "assets"）和相对路径（如 "textures/hero.png"），拼接成物理路径。如果路径不包含 ://，直接检查是否存在，存在则返回绝对路径。若均失败，返回 nullptr，日志记录错误。

```cpp
// 使用逻辑路径加载，
std::string logicalPath = "assets://textures/hero.png";
auto texture1 = resMgr.getRes<TextureResource>(logicalPath);
if (texture1)
{
    renderer->draw(texture1);
}

// 或者直接使用物理路径
std::string physicalPath = "C:/data/file.png";
auto texture2 = resMgr.getRes<JsonResource>(physicalPath);
if (texture2)
{
    renderer->draw(texture2);
}
```

注意：资源类型必须继承自 ResourceBase。

### 9.4.5 重新加载资源

当文件发生变化时（如热更新），可强制重新加载，此时使用 reload() 。这将清除指定资源的缓存，重新从磁盘加载。

```cpp
// 假设用户修改了着色器文件后重新加载
std::string logicalPath = "assets://shaders/effect.glsl";
resMgr.reload(logicalPath);
```

### 9.4.6 资源回收

使用 unloadUnused() 卸载无引用的资源，这将遍历所有缓存资源，释放那些外部引用计数为 1（仅缓存持有）的资源。适合定期调用，防止内存膨胀。

```cpp
// 在每帧或每几分钟调用
resMgr.unloadUnused();
```

### 9.4.7 清空所有资源

使用 clearAll() 清除所有挂载点、加载器和缓存资源，其中init() 和 shutdown() 内部调用的就是这个函数。

```cpp
resMgr.clearAll();
```

### 9.4.8 手动路径解析

若需要将逻辑路径转换为物理路径而不加载资源，可使用 resolvePath()：

```cpp
std::string logicalPath = "assets://config.json";
auto physPath = resMgr.resolvePath(logicalPath);
if (physPath)
{
    std::cout << "Physical path: " << *physPath << std::endl;
}
```

### 9.4.9 清理与关闭

使用 clearAll() 清理所有资源。析构函数内会自动调用此函数。

```cpp
resMgr.shutdown();
```

## 9.5 完整示例

```cpp
// TextureResource.hpp
#pragma once
#include "Resource/virResourceBase.hpp"
#include <string>

class TextureResource : public core::ResourceBase
{
public:
    bool loadFromFile(const std::filesystem::path& path)
    {
        // 模拟加载纹理
        mPath = path.string();
        return true;
    }

    void bind() const
    {
        /* OpenGL 绑定代码 */
    }

private:
    std::string mPath;

};
```
```cpp
// JsonResource.hpp
#pragma once
#include "Resource/virResourceBase.hpp"
#include <nlohmann/json.hpp>

class JsonResource : public core::ResourceBase
{
public:
    bool parse(const std::filesystem::path& path)
    {
        std::ifstream f(path);
        if (!f.is_open()) return false;
        f >> mJson;
        return true;
    }

    const nlohmann::json& get() const
    {
        return mJson;
    }

private:
    nlohmann::json mJson;

};
```
```cpp
// main.cpp
#include "Resource/lgcResourceManager.hpp"
#include "TextureResource.hpp"
#include "JsonResource.hpp"
#include <iostream>

int main()
{
    auto& resMgr = core::ResourceManager::get();
    resMgr.init();

    // 1. 挂载虚拟路径
    resMgr.mount("assets", "./game_assets");
    resMgr.mount("config", "./configs");

    // 2. 注册加载器
    resMgr.registerLoader(".png", [](const std::filesystem::path& p) -> std::shared_ptr<core::ResourceBase> {
        auto tex = std::make_shared<TextureResource>();
        return tex->loadFromFile(p) ? tex : nullptr;
    });

    resMgr.registerLoader(".json", [](const std::filesystem::path& p) -> std::shared_ptr<core::ResourceBase> {
        auto json = std::make_shared<JsonResource>();
        return json->parse(p) ? json : nullptr;
    });

    // 3. 获取并使用资源
    auto tex = resMgr.getRes<TextureResource>("assets://sprites/player.png");
    if (tex) tex->bind();

    auto config = resMgr.getRes<JsonResource>("config://app.json");
    if (config)
        std::cout << config->get().dump(2) << std::endl;

    // 4. 热更新示例
    std::thread([&]{
        std::this_thread::sleep_for(std::chrono::seconds(10));
        resMgr.reload("assets://shaders/effect.glsl"); // 假设注册了 .glsl 加载器
    }).detach();

    // 5. 定期清理无引用资源
    while (true) {
        std::this_thread::sleep_for(std::chrono::minutes(5));
        resMgr.unloadUnused();
    }

    return 0;
}
```

## 9.6 线程安全说明

读操作（getRes() 、resolvePath()）使用共享锁，可并发执行。

写操作（mount() 、registerLoader() 、reload() 、unloadUnused()、clearAll()）使用独占锁。

资源加载过程（loadFromDisk() 调用用户 loader()）不持有资源锁，允许并发加载不同文件。

注意：用户提供的 loader 函数需自身保证线程安全（若可能被多线程同时调用）。

## 9.7 注意事项
* 资源类型要求：所有资源类必须继承自 core::ResourceBase，否则无法存入 mResources（shared_ptr<ResourceBase> 要求可转换）。建议资源类提供合适的构造、析构和移动语义。
* 加载器线程安全：加载器可能被多个线程同时调用（如多个 getRes 同时首次加载不同文件），需确保加载器代码线程安全。若加载器内部需访问共享资源（如 GPU 上下文），应外部同步。
* 缓存键：缓存键是物理路径的字符串。不同逻辑路径若映射到同一物理文件，会共享同一资源实例。路径区分大小写，取决于文件系统。
* 引用计数与 unloadUnused：unloadUnused 检查 use_count() == 1，即仅缓存持有该资源时释放。若外部一直持有 shared_ptr（如场景中的对象），资源不会释放。切勿在资源内部存储指向自身的 shared_ptr（循环引用），否则永远不会被回收。
* 别名重复：同一别名重复挂载会覆盖旧映射（日志将会记录信息）。
* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 性能：getRes 在缓存命中时仅一次共享锁查找，非常快。首次加载会在写锁下插入缓存，但加载过程本身（调用 loader）在锁外执行，不影响并发。reload 执行写锁并重新加载，应避免频繁调用。
