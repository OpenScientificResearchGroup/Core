# 6. 语言管理器

## 6.1 简介

语言管理器（I18n Manager）是一个基于 JSON 的国际化（i18n）管理单例类，支持多语言翻译、语言切换、回退机制、fmt 格式化参数以及简单的复数支持。它采用读写锁保证线程安全。

## 6.2 特性

* **多语言支持**：可加载多个语言包，按需切换当前语言；
* **回退机制**：若当前语言缺少某键，自动回退到指定的回退语言（一般是 "en-US"）；
* **JSON 加载**：从 JSON 文件或字符串加载翻译数据，支持嵌套键（如 "user.welcome"）；
* **fmt 格式化**：翻译字符串支持 {arg} 占位符，使用 fmt::arg 传参；
* **复数支持**：简易复数规则（_one/_other 后缀），可扩展为 CLDR 规则；
* **线程安全**：使用 std::shared_mutex 实现读多写少场景的高效并发。

## 6.3 使用方法

### 6.3.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 I18n Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	I18nManager::get().init(ConfigManager::get().getValue<std::string>("/Core/i18n/locale"));
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
// 获取单例
core::I18nManager& i18n = core::I18nManager::get();
// 初始化
std::string locale = "zh-CN"; // 初始语言代码，如 "zh-CN"、"en-US"
std::string fallbackLocale = "en-US"; // 回退语言，默认为 "en-US"
bool isSucceeded = init(locale, fallbackLocale);
```

### 6.3.2 加载语言包

支持从 JSON 文件或 JSON 字符串加载翻译数据，使用 loadFromFile() 从文件加载语言，使用 loadFromString() 从字符串中加载语言。

```cpp
// 从文件中加载
std::string path = "lang/zh-CN.json";
std::string locale = "zh-CN";
i18n.loadFromFile(path, locale);

// 从字符串加载
std::string data; // 不知道从哪里冒出来的字符串
std::string locale = "en-US";
i18n.loadFromFile(data, locale);
```

### 6.3.3 切换语言

使用 setLocale() 切换语言

```cpp
// 切换当前语言为 locale
std::string locale = "en-US";
i18n.setLocale(locale);
```
注意：当前实现仅设置语言标识，不会自动加载语言包（需外部先加载好）。

### 6.3.4 获取当前语言、回退语言名称

使用 getLocale() 获取当前语言名称，使用 getFallbackLocale() 获取回退语言名称

```cpp
std::string locale =  i18n.getLocale();          // 获取当前语言名称
std::string fallbackLocale = i18n.getFallbackLocale();  // 获取回退语言名称
```

### 6.3.5 翻译函数

使用 t() 获取语言。

```cpp
// 无参数
std::string ok = i18n.t("common.ok");  // "确定"

// 命名参数（推荐，更清晰）
std::string msg = i18n.t("user.welcome", fmt::arg("name", "张三"));
// 若当前语言为 zh-CN: "欢迎，张三！"
// 若当前语言为 en-US: "Welcome, {name}!" (需加载对应语言包)

// 位置参数
std::string countMsg = i18n.t("file.count", fmt::arg("count", 5));
// "共 5 个文件"
```

注意：查找顺序为：
1. 当前语言中查找键；
2. 若未找到，在回退语言中查找；
3. 若都未找到，返回 key 本身作为兜底。

### 6.3.6 复数翻译

使用 tc() 获得复数翻译。

```cpp
std::string msg1 = i18n.tc("file.count", 1, fmt::arg("count", 1));   // "1 个文件"
std::string msg5 = i18n.tc("file.count", 5, fmt::arg("count", 5));   // "5 个文件"
std::string msg0 = i18n.tc("file.count", 0, fmt::arg("count", 0));   // "0 个文件" (使用 _other)
```

复数规则：
1. 若 count == 1，尝试 baseKey + "_one"；
2. 否则（包括 0 和大于 1），尝试 baseKey + "_other"；
3. 若特定后缀不存在，回退到 baseKey。

注意：此复数规则仅考虑了建议的英语风格，实际工业级项目应引入更加复杂的 CLDR 复数规则。

### 6.3.7 管理语言包

使用 clear() 清除所有翻译：

```cpp
i18n.clear();
```

使用 removeLocale() 移除指定语言的翻译数据：

```cpp
std::string locale = "zh-CN";
i18n.removeLocale(locale);
```

注意：若移除的是当前语言且还有其它语言，自动切换到回退语言（日志记录）。

### 6.3.8 清理与关闭

使用 shutdown() 清空所有翻译数据，重置当前语言和回退语言，析构函数将自动调用 shutdown() 。

```cpp
i18n.shutdown();
```

## 6.4 语言文件格式要求

示例 JSON 文件（zh-CN.json）如下：

```json
{
    "common.ok": "确定",
    "common.cancel": "取消",
    "user.welcome": "欢迎，{name}！",
    "file.count": "共 {count} 个文件",
    "file.count_one": "1 个文件",
    "file.count_other": "{count} 个文件"
}
```

有以下格式要求：
* 根对象必须为 JSON 对象（{}）。
* 键为翻译键（如 "common.ok"），值为字符串。

注意，类型支持自动转换：数字、布尔值会转为字符串，复杂类型会转为 JSON 字符串并记录警告。

## 6.5 完整示例

假设我们已经准备好了这两个文件：

lang/en-US.json：

```json
{
    "common.ok": "OK",
    "common.cancel": "Cancel",
    "user.welcome": "Welcome, {name}!",
    "file.count": "{count} file(s)",
    "file.count_one": "1 file",
    "file.count_other": "{count} files"
}
```

lang/zh-CN.json：

```json
{
    "common.ok": "确定",
    "common.cancel": "取消",
    "user.welcome": "欢迎，{name}！",
    "file.count": "{count} 个文件",
    "file.count_one": "1 个文件",
    "file.count_other": "{count} 个文件"
}
```

使用代码

```cpp
#include "I18n/lgcI18nManager.hpp"
#include <iostream>

int main()
{
    auto& i18n = core::I18nManager::get();

    // 初始化
    i18n.init("en-US", "en-US");

    // 加载语言包
    i18n.loadFromFile("lang/en-US.json", "en-US");
    i18n.loadFromFile("lang/zh-CN.json", "zh-CN");

    // 测试英文
    std::cout << i18n.t("common.ok") << std::endl;                          // OK
    std::cout << i18n.t("user.welcome", fmt::arg("name", "Alice")) << std::endl; // Welcome, Alice!
    std::cout << i18n.tc("file.count", 1, fmt::arg("count", 1)) << std::endl;     // 1 file
    std::cout << i18n.tc("file.count", 5, fmt::arg("count", 5)) << std::endl;     // 5 files

    // 切换到中文
    i18n.setLocale("zh-CN");

    std::cout << i18n.t("common.ok") << std::endl;                          // 确定
    std::cout << i18n.t("user.welcome", fmt::arg("name", "张三")) << std::endl; // 欢迎，张三！
    std::cout << i18n.tc("file.count", 1, fmt::arg("count", 1)) << std::endl;     // 1 个文件
    std::cout << i18n.tc("file.count", 5, fmt::arg("count", 5)) << std::endl;     // 5 个文件

    // 测试回退：假设当前语言没有 "common.exit"，但回退语言有
    std::cout << i18n.t("common.exit", fmt::arg("name", "Test")) << std::endl; // "common.exit" (回退到键本身)

    return 0;
}
```

## 6.6 线程安全
读操作（t() 、 tc() 、getLocale() 、hasKey()）使用共享锁，可并发执行。

写操作（init() 、 shutdown() 、 loadFromXxx() 、 setLocale() 、 removeLocale() 、 clear()）使用独占锁。

注意：t 内部调用 getRaw() 持有读锁，但 fmt::vformat 是在锁外执行的，避免阻塞。

## 6.7 注意事项

* 格式化参数类型：fmt::arg 创建的命名参数需与翻译字符串中的 {name} 占位符匹配。也可以使用位置参数：fmt::arg("0", value) 对应 {0}，但推荐命名参数；
* 复数规则的局限性：当前复数规则仅为示例，仅适用于英语（1 用 _one，其余 _other）。对于其他语言（如中文无复数变化、俄语有复杂规则），需扩展为完整的 CLDR 规则引擎；
* 键的命名规范：建议使用点分隔的层级命名，如 "user.welcome"、"error.file.not_found"。避免使用特殊字符（除点和下划线外），以免与格式化冲突；
* 字符串视图的生命周期：getRaw() 返回的 std::string_view 指向内部存储的字符串，在 mMutex 保护下有效。若在持有锁期间使用，安全；若在锁外使用，需注意字符串可能被修改或移除（removeLocale() 、 clear()）。当前 t 内部调用 getRaw() 并立即格式化，不会延长锁持有时间；
* 日志：必须在使用前确保日志管理器（Log Manager）已经初始化，或注释掉相关的日志函数；
* 线程安全：t 和 tc 内部使用读锁，格式化在锁外，适合高并发读取；setLocale 使用写锁，但不会自动重新加载语言包，需外部确保语言包已加载。
* 语言包热更新：若要支持运行时重载语言包，可调用 loadFromFile 重新加载指定语言。加载后已有 std::string_view 可能失效，但 t 总是重新查找，无影响。
