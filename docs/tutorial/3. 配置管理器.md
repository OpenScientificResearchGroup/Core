# 3. 配置管理器

## 3.1 简介

配置管理器（Config Manager）是一个基于 JSON 的分层配置管理模块，采用单例模式，提供线程安全的配置读写、注册默认值、用户配置持久化等功能。

## 3.2 特性

* **分层优先级**：用户配置 > 插件注册默认值 > 硬编码兜底 > 类型默认构造；
* **路径风格**：类 Windows 注册表风格，使用 / 分隔，例如 /plugins/git/timeout；
* **自动持久化**：修改配置后立即写入用户配置文件；
* **类型安全**：模板接口自动处理类型转换；
* **线程安全**：面向读多写少的场景，采用读写锁（std::shared_mutex）支持并发读/写；
* **元数据支持**：每个配置项可携带描述、默认值，便于生成 UI。

## 3.3 使用方法

### 3.3.1 初始化

在程序启动时，lgcCore.cpp 中的 initializeCore() 会自行调用 Config Manager 的 init() 方法：

```cpp
// lgcCore.cpp
void initializeCore(
	const std::string& appName,
	const std::string& confPath,
	const std::string& coreConfPath,
	const std::string& corePluginsDir,
	const std::string& userPluginsDir
)
{
    // ......
	ConfigManager::get().init(confPath, coreConfPath);
    // ......
}
```

由于几个管理器之间有依赖关系，初始化的顺序不能被随意改变，因此**在组合使用时不推荐自行调用 init()** （除非你真的非常了解你在做什么！）。

当然如果想要单独拿出来用也是可以的，记得去掉其中的 Log Manager 避免出现编译错误。此时自行调用 init() 进行初始化。

```cpp
ConfigManager::get().init(
    /*在此处传入一个 std::string ，表示用户自定义配置的 json 存放位置*/,
    /*在此处传入一个 std::string ，表示 Core 默认配置的 json 存放位置*/
    );
```

### 3.3.2 注册配置项（插件/模块）

有两种方式注册配置项，分别是从 json 文件中读取或在插件中一个一个单独加入。

如果想要从 json 中读取，请使用 readDefaultValue() 。

如果想要插件加载时写入，则使用 registerOption() 向管理器注册自己的配置项元数据。注册的信息会存储在内存的 Schema 层，用于提供默认值和描述。

```cpp
auto& cfg = core::ConfigManager::get();
// 从 json 文件中直接读取
readDefaultValue("test.json");

// 注册一个整数类型配置项
cfg.registerOption<int>("/plugins/git/timeout", 30, "Git operation timeout in seconds");

// 注册字符串类型
cfg.registerOption<std::string>("/ui/theme", "light", "UI theme (light/dark)");

// 注册复杂 JSON 对象
nlohmann::json defaultFilters = {{"enabled", true}, {"patterns", {"*.tmp", "*.log"}}};
cfg.registerOption("/tools/cleaner/filters", defaultFilters, "File filter settings");
```

在使用 registerOption() 时，请注意以下参数：
* key：以 / 开头的完整路径，如 /section/key；
* defaultValue：任意能被 nlohmann::json 接受的值；
* description：描述文本（用于 UI 显示或文档）。

### 3.3.3 读取配置值

使用 getValue<T> 获取配置值，遵循分层回退规则：

```cpp
auto& cfg = core::ConfigManager::get();
int timeout = cfg.getValue<int>("/plugins/git/timeout");
std::string theme = cfg.getValue<std::string>("/ui/theme");

// 可提供硬编码兜底值（仅当用户未设置且无注册默认值时生效）
bool featureEnabled = cfg.getValue<bool>("/experimental/feature", false);
```

返回值采用以下策略：
1. 若用户配置文件中存在该键，返回用户值；
2. 若不存在，返回注册时的默认值（Schema 层）；
3. 若未注册，返回传入的 hardFallback（如果有）；
4. 若以上均无，返回 T()（例如 int 返回 0）。

### 3.3.4 修改配置值

使用 setValue 修改配置，修改会立即保存到用户配置文件：

```cpp
auto& cfg = core::ConfigManager::get();
cfg.setValue("/plugins/git/timeout", 60);
cfg.setValue("/ui/theme", "dark");
```

注意：修改仅影响用户配置层，不会修改注册的默认值（Schema 层）。

### 3.3.5 重置配置

若想删除用户设置，恢复为默认值，调用 reset：

```cpp
auto& cfg = core::ConfigManager::get();
cfg.reset("/plugins/git/timeout");
```

之后再次 getValue 将返回注册时的默认值（或兜底值）。

### 3.3.6 获取配置描述

```cpp
auto& cfg = core::ConfigManager::get();
const std::string& desc = cfg.getDescription("/plugins/git/timeout");
```

返回注册时传入的描述文本，若键不存在则返回空字符串。

### 3.3.7 获取所有已注册的 Schema

```cpp
auto& cfg = core::ConfigManager::get();
auto schema = cfg.getSchema();
for (const auto& [key, option] : schema)
    std::cout << key << " : " << option.description << "\n";
```

ConfigOption 结构包含 key、defaultValue、description、typeName。

### 3.3.8 调试输出

输出当前用户配置的 JSON 字符串（带缩进）：

```cpp
auto& cfg = core::ConfigManager::get();
std::cout << cfg.dump() << std::endl;
```

### 3.3.9 关闭与清理

在程序退出前可调用 shutdown 释放内部资源（非必须，单例随进程销毁自动清理）：

```cpp
auto& cfg = core::ConfigManager::get();
cfg.shutdown();
```

## 3.4 默认配置文件格式

defaults.json 文件格式如下， readDefaultValue() 会递归遍历并自动注册每个包含 "value" 的节点：

```json
{
    "Core": {
        "log": {
            "path": {
                "value": "./logs/core.log",
                "description": "Path to the core log file."
            },
            "max_file_size": {
                "value": 5242880,
                "description": "Maximum size of the core log file in bytes."
            },
            "max_files": {
                "value": 3,
                "description": "Maximum number of core log files to keep."
            },
            "console_out": {
                "value": true,
                "description": "Enable or disable console output for core logs."
            }
        },
        "command": {
            "max_history": {
                "value": 50,
                "description": "Maximum number of commands to keep in history."
            }
        },
        "task": {
            "num_threads": {
                "value": 8,
                "description": "Maximum number of threads for task execution."
            }
        },
        "i18n": {
            "locale": {
                "value": "zh-cn",
                "description": "Default language for the application."
            }
        }
    }
}
```

有以下格式要求：
* 每个配置项必须是一个对象，且包含 "value" 键
* 可选的 "description" 键提供描述
* 对象嵌套即表示路径层级，例如 /Core/log/path

## 3.5 完整示例

```cpp
#include <iostream>
#include "Config/lgcConfigManager.hpp"

int main()
{
    auto& cfg = core::ConfigManager::get();

    // 初始化
    if (!cfg.init("user.json", "defaults.json"))
    {
        std::cerr << "Failed to init config manager\n";
        return 1;
    }

    // 从文件中注册
    readDefaultValue("test.json");

    // 直接注册
    cfg.registerOption<int>("/plugins/git/timeout", 30, "Git timeout");
    cfg.registerOption<std::string>("/plugins/git/editor", "vim", "Git editor");

    // 读取配置
    int timeout = cfg.getValue<int>("/plugins/git/timeout");
    std::string editor = cfg.getValue<std::string>("/plugins/git/editor");
    std::cout << "timeout: " << timeout << ", editor: " << editor << std::endl;

    // 修改配置
    cfg.setValue("/plugins/git/timeout", 60);

    // 再次读取，验证修改
    timeout = cfg.getValue<int>("/plugins/git/timeout");
    std::cout << "new timeout: " << timeout << std::endl;

    // 输出当前用户配置的 JSON 字符串（带缩进）
    std::cout << cfg.dump() << std::endl;

    // 重置
    cfg.reset("/plugins/git/timeout");
    timeout = cfg.getValue<int>("/plugins/git/timeout");  // 回到30
    std::cout << "after reset: " << timeout << std::endl;

    return 0;
}
```

## 3.6 线程安全

所有公有成员函数都是线程安全的：

getValue()、getDescription()、getSchema()、dump() 使用共享锁，允许多个线程并发读取。

registerOption()、setValue()、reset()、init()、shutdown() 使用独占锁，保证修改时数据一致。

## 3.7 注意事项
* 所有键名必须使用 / 作为分隔符，且以 / 开头。如果想要修改，请在 lgcConfigManager.hpp 中修改 SEPARATOR 该项。
* 路径中的空格和特殊字符由 JSON 库处理，但作为键时应尽量避免。
* 用户配置文件将在每次使用 setValue() 后立即保存，若频繁修改可能影响性能。
* 若需要支持配置热加载，请重新调用 load()，并配合 Event Manager 进行广播。
